📅 27 Mar 2026 — Day 11: Platform Globalization & Specialty Standardization
----------------------------------------------------
• Globalized the platform by integrating country, state, and city fields
    across the user model, APIs, and all related frontend/backend logic
  • Replaced raw text specialty input with a structured dropdown of predefined
    specialty options throughout the platform
  • Reworked doctor verification workflow to a two-step process:
    - Step 1: NMC API runs auto-verification on every profile update, setting
      an nmc_verified flag (TRUE/FALSE)
    - Step 2: Admin can only approve or reject a doctor after NMC verification
      has passed — doctors with a FALSE nmc_verified status are blocked from
      admin approval
    - Final doctor status (VERIFIED) is only set after both NMC check passes
      and admin explicitly approves

📅 30 Mar 2026 — Day 13: NMC Import, Rate Limiting & Redis Cache
----------------------------------------------------
  • Built "Import doctors from NMC site" feature based on registration year:
    - Fetches doctor records directly from the NMC site for a given year
    - Only one import request allowed per day to avoid overloading the NMC
      site — enforced at the API level
    - Request count and the last used offset per year are saved to the DB,
      so subsequent imports for the same year skip already-fetched records
      and only pull new unique doctor entries
  • Implemented Redis cache for the doctor list endpoint to reduce DB load
    on repeated queries
